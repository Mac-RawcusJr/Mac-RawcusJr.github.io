Simple Problem

During an open class coding activity during my EDA course I was struggling with one of the most simple questions and felt really frustrated with myself during the whole process and after I figured out the simple answer.

This was one of the first question when I had to give a value to a variable and usually when you first create a variable you give it some sort of value whether it be a number or words or even a sentence. However, this tasks was to assign value to these variables without editing the code around the variables themselves. I originally thought that It was a trick question and therefore, edited the variables themselves to give the values. But then I thought no it's not a trick question and tried to figure out how to change the values outside of the variables. This is where the frustration began.

I spent so long trying to figure out how to change the values with complex code that I was researching and still couldn't past the test. The problem with this situation is that I over thought everything and missed the simple solution. I also wasted a lot of time and got frustrated, unable to do much other work. I learnt from this experience that I have to slow down and keep things simple while also understanding the problems better.

Elegant problem Solving

Theres was a day where wsa tasked with taking the recycling to the recycling center to get rid of it and this task would take roughly an hour. So I thought to myself that I would put on a load of my washing before I left so that when I got back I could hang it out. And this is where my problems began.

To do my washing I had to put my clothes in the machine which was full, so I had to put those clothes into a basket which was also full of folded sheets and rugs. So to do these away I would put them on the shelf in the laundry but at the time needing new carpet being put down so the shelf was in the middle of the laundry. So instead this lead to having to do the carpet, to move the shelving, to put away the sheets and rugs to empty the washing machine to do my own washing so I could go and do the recycling.

I found this to be elegant problem solving because I did not overthink any situation and when I came across a problem I figured out how to fix it and continue on with my task. I figured out my goal (to put washing on) and discovered the tasks I needed to do (unload machine, put sheets away etc, etc) and one by one I completed the tasks and the goal. So thought out the process I didn’t feel frustrated or rushed and I was confident in my abilities to solve each problem. From this exchange I learnt that as you work towards your goal it is best to solve each problem one at a time and go with the flow while maintaining and visualising your final goal.

Control Flow and Loops

Control flow is the sequence at which the conmputer reads the code and in what order they are out into action. the default Control flow order is typically from top to bottom until you write out specific code that tells the computer to prioritize this bit of code over some others.

Changing the Control Flow seems to mostly happen when we use Loops. Loops are pieces of code the instruct the computer to repeat a certain peice of code until instructed to stop or the result becomes false. We use loops for various situations but the most simple way is to instruct a computer to comb through arrays of information and can perform a step by step process of dealing with that information.

Loops are like your morning routine and the activator is your alarm, your alarm goes off which instructs you to wake up, then shower, get dressed, have breakfast, then head to work. its a consistant loop that when activiated performs a series of actions. And control flow is a more broader version of loops with less routine.

The DOM

The Document Object Model is a translated and simplified diagram of a Webpages HTML element/object hierarchy. You can think of the DOM as a translation of html so that other languages such as JavaScript and CSS can understand it better, or vice versa.

Css Positioning

What is the position property?

The position property manipulates an elements position within the webpage.

Relative

According to W3 School relative positioning “is positioned relative to its normal position.” The positioning allows an element to be positioned relative to its default spot. So, you can move an element in direction any amount of distance and not be affected by many other elements. However, this position style won’t work without further coding giving the element instructions on where to move to. This means that the top, right, bottom, and left

Fixed

A Fixed position makes the elements position relative to the viewpoint of the web page. In simple terms this positioning makes an element stay still on a screen even if you are scrolling and moving around on the actual webpage the element will stay in the same place. An example of this is typical a menu option on a page that stays tucked away in a corner available for use no matter where you go on a page.

Absolute

Relative positioning is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed)

This diagram provides a visual example of how each position affects an element. As we can see a relative position is adjusted from its default position. An absolute position is determined by the distance from margins but still can be scrolled passed unlike and a fixed position which is also determined by the distance from margins but will stay in the spot even while scrolling.
